# Jimeng Free API Windows 便携一键启动包设计

## 目标

为 Windows 10/11 x64 用户制作一个免安装、免 Docker 的 ZIP 便携包。用户解压后双击中文批处理文件即可启动、停止服务或查看日志，不需要预装 Node.js、npm 或编译工具。

## 交付范围

交付物为 `jimeng-free-api-windows-x64-portable.zip`，解压后的顶层目录提供：

- `启动服务.bat`：启动服务、检查启动结果并打开管理页面。
- `停止服务.bat`：只停止由本便携包启动的服务。
- `查看日志.bat`：打开当前运行日志。
- `使用说明.txt`：说明首次使用、端口修改、数据位置和常见故障。
- `config\portable.env`：保存用户可修改的端口等便携包配置。
- `runtime\`：固定版本的官方 Windows x64 Node.js 运行时。
- `app\`：上游项目构建产物、生产依赖、静态资源和服务配置。
- `data\`：持久化 SQLite 数据库、管理员密码、统计数据和媒体记录。
- `logs\`：启动器输出和应用日志。
- `run\`：运行期间创建 PID 等状态文件；停止后清理。

ZIP 中不包含源码仓库、开发依赖、npm 缓存或构建工具。

## 架构与运行流程

构建阶段使用上游锁文件安装依赖并执行 `npm run build`，再只保留生产依赖。由于项目依赖原生模块 `better-sqlite3`，生产依赖必须在 Windows x64、与便携包 Node.js 相同的主版本下安装并进行一次加载验证。

运行阶段由 PowerShell 启动器承担可靠的进程管理，中文 `.bat` 文件只作为双击入口：

1. 将工作目录固定为便携包根目录，确保中文、空格路径可用。
2. 读取 `config\portable.env`；默认端口为 `8001`。
3. 检查 Node.js 运行时、应用文件及端口占用。
4. 设置 `DB_PATH=<根目录>\data\jimeng.db`、`SERVER_PORT`、`SERVER_HOST` 和 `SERVER_ENV=portable`。
5. 以 `app\` 为工作目录后台启动 `runtime\node.exe dist\index.js`，将标准输出和错误写入 `logs\service.log`。
6. 将进程 ID 写入 `run\service.pid`。
7. 在有限时间内轮询 `http://127.0.0.1:<端口>/`；成功后打开默认浏览器。
8. 若进程提前退出或健康检查超时，清理失效 PID，并在窗口中显示日志位置和末尾错误。

便携包携带专用的 `app\configs\portable\` 配置：临时目录和日志目录指向包内持久化位置，静态资源仍指向 `app\public\`。端口和监听地址通过上游已经支持的 `SERVER_PORT`、`SERVER_HOST` 环境变量覆盖，不修改上游源码。这样用户修改 `portable.env` 后，下次启动即可生效。

## 进程与数据边界

- 启动器通过 PID 文件记录自己创建的进程，不按进程名批量终止 Node.js，避免影响用户的其他程序。
- 再次启动时，若 PID 仍存活且对应本包应用，则提示服务已运行并打开管理页。
- 若 PID 文件失效，则自动清理后重新启动。
- `data\`、`logs\`、`config\portable.env` 均位于便携包根目录，整体移动后仍可运行。
- 更新程序时应保留上述三个位置，以延续管理员配置和历史数据。

## 配置

`config\portable.env` 首版仅暴露：

```text
PORT=8001
HOST=0.0.0.0
AUTO_OPEN_BROWSER=1
```

端口必须为 `1` 到 `65535` 的整数；主机地址不能为空；`AUTO_OPEN_BROWSER` 仅接受 `0` 或 `1`。非法配置会阻止启动并显示具体字段。

## 错误处理

- 文件缺失：列出缺失路径，提示重新解压完整 ZIP。
- 端口占用：显示端口号，不擅自结束占用进程。
- 原生模块不兼容：构建验证直接失败，不生成交付 ZIP。
- 服务启动失败：显示日志末尾内容并保留完整日志。
- 停止失败：仅在 PID 对应本包应用时尝试结束进程；无法确认身份时不终止进程。
- 路径问题：所有脚本使用绝对路径和严格引用，覆盖中文及空格目录。

## 测试与验收

构建阶段必须完成：

1. `npm run build` 成功。
2. 使用内置 Node.js 加载 `better-sqlite3` 成功。
3. 便携包中不存在 `src\`、开发依赖和 npm 缓存。
4. ZIP 可正常解压，关键文件齐全。

运行阶段必须完成：

1. 在全新临时目录启动，管理页返回成功响应。
2. 默认浏览地址为 `http://localhost:8001`。
3. 二次启动不会产生重复服务进程。
4. 停止脚本能终止本包服务并清理 PID。
5. 停止后重启成功，`data\jimeng.db` 保持存在。
6. 修改端口后可在新端口启动。
7. 端口被占用时明确失败且不结束占用者。
8. 在含中文和空格的目录中完成启动、访问和停止。

生成式图片与视频接口需要用户自己的即梦 Session ID，不作为离线打包验收条件；验收只验证服务、控制台、模型兜底列表以及数据持久化。

## 非目标

- 不制作安装器或 Windows 服务。
- 不支持 Windows ARM64、Windows 7、macOS 或 Linux。
- 不内置用户 Session ID 或管理员密码。
- 不修改上游业务接口和控制台功能。
- 不自动联网升级应用。
